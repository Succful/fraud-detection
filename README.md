---

## 📋 О проекте

Проект реализует систему фрод-детекции на основе реальных банковских транзакций. Система выявляет подозрительных клиентов через комбинацию **rule-based флагов** и **машинного обучения**, присваивая каждому клиенту итоговый статус риска.

### Ключевые результаты

| Метрика | Значение |
|---|---|
| Объём датасета | 13,385,915 транзакций |
| Количество клиентов | 1,219 |
| Флагов фрода | 9 |
| ML моделей | 5 |
| Лучший AUC | 0.739 (Logistic Regression) |
| Выявлено фрод-клиентов | ~3.2% от базы |

## 🚩 Описание флагов

### Flag 1 — Velocity
Клиент совершает более 3 транзакций в день, средний чек выше обычного на 34%+, при этом транзакции происходят в 2+ разных штатах. Признак карточного мошенничества с множественным использованием.

### Flag 2 — Bad Merchant
Клиент совершает транзакции у мерчантов, у которых более 40% операций завершаются с ошибкой. Такие мерчанты могут быть скиммерами или фиктивными терминалами.

### Flag 3 — Night TX
Средний чек ночных транзакций (00:00–05:00) превышает дневной в 3 раза. Нетипичное поведение, характерное для несанкционированного использования карты.

### Flag 4 — Fast TX
Две крупные транзакции (каждая >5x от среднего) с интервалом менее 4 минут. Классический паттерн тестирования украденной карты.

### Flag 5 — Online Spike
Средняя сумма онлайн-транзакций превышает офлайн в 2 раза. Признак использования скомпрометированных данных карты в интернете.

### Flag 6 — Weekend Binge
В выходные дни клиент тратит в 2.5x больше чем в будни. Нетипичное поведение, особенно значимое в сочетании с другими флагами.

### Flag 7 — High Unique Merchants
Клиент входит в топ 5% по количеству уникальных мерчантов. Аномально высокое количество посещённых точек может указывать на карусельное мошенничество.

### Flag 8 — Amount Spike
Как минимум одна транзакция превышает личный средний чек клиента в 10 раз. Резкий выброс суммы — сигнал несанкционированной операции.

### Flag 9 — Cross State
В течение одного месяца транзакции зафиксированы в 4 и более штатах. Географически невозможное или крайне подозрительное поведение.

## ⚙️ Установка и запуск

### 1. Клонировать репозиторий
```bash
git clone https://github.com/ТВО_ИМЯ/fraud-detection.git
cd fraud-detection
```

### 2. Установить зависимости
```bash
pip install -r requirements.txt
```

### 3. Скачать датасет
Датасет доступен на Kaggle: [Credit Card Transactions](https://www.kaggle.com/)

Положи файл по пути:
```
data/transactions_data.csv
```

### 4. Обновить путь в ноутбуке
В **Блоке 3** замени путь:
```python
PATH = r'data/transactions_data.csv'
```

### 5. Запустить ноутбук
```bash
jupyter notebook fraud_detection_full.ipynb
```
Затем: **Cell → Run All**
